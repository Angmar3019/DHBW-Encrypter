Ver-/Entschluesseln von Texten
- Beschränkung auf ASCII-Zeichen 32-126

- Menü zur Auswahl des Algorithmus (Terminal)
	- Caesar
	- Trithemius
	- Vigenere
	- One-Time-Pad
	- RSA
	- (Morse?) -> weiter beschränkt auf Buchstaben
	
- Wahl zwischen Ver- und Entschlüsseln
	- (Kann bei Morse Übersprungen werden)
	
- Menü zur Auswahl der Eingabeart
	- Direkteingabe über Terminal
	- Dateiname im aktuellen Verzeichnis (txt)
	- absoluter Dateipfad für externe Dateien (txt)

- Sequentielle Abfrage von nötigen Inputs
	- Caesar: Verschiebung: Integer
	- Vigenere: Schluesseltext/-wort 
	- Trithemius: kein Input
	- RSA: 2 Primzahlen (p, q, p ungleich q, p * q > 256^2), Schlüssel e/d (Programm liefert valide Beispiele oder User gibt eigenen Wert an)   
	- One-Time-Pad: Schluesseltext/-wort ODER: zufällig generiert, damit User selbst nichts überlegen/tippen muss (wird im Modul übernommen, im Menü nur Abfrage)
	- (Morse: kein Input)
	
Potenzielle Erweiterungen:
- "Prüfmodus": Prüfsumme/-hash des Klartextes wird erstellt und ausgegeben -> Vergleich von Ursprungstext und entschlüsseltem Text
- Erweiterung auf Bilddateien
	- Umwandlung von Bild zu Text (oder vice versa), Anwendung der bereits implementierten Textverschlüsselungen
	- Implementieren von Bildverschlüsselungen (z.B Kodieren/Auslesen von Nachrichten in Bildern)
